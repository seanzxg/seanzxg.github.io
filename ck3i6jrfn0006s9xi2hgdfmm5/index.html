关于几个指令的解读

  1. yarn dev => cross-env NODE_ENV=development nodemon server/index.js --watch server

    cross-env`:是运行跨平台设置和使用环境便利那个的脚本。详情

    nodemon : Nodemon is a utility that will monitor for any changes in your source and automatically restart your server。来点英文你懂得，就是刷新服务的一个node工具 而且nuxt环境自带，无需过度关心。

  2. yarn generate 静态文件站点生成，支持一些基础的路由，但是静态文件不涉及网络请求和服务端的一些操作，仅作为静态网站文件。

  3. yarn start => cross-env NODE_ENV=production node server/index.js

    yarn start需要先进行yarn build构建

    在服务器运行是运行server/index.js，资源取自.nuxt文件下
